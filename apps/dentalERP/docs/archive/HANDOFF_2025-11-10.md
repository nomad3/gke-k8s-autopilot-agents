## THE CORE ISSUE - NetSuite API

### Problem Statement

**Goal:** Sync financial data from NetSuite per subsidiary automatically

**Current Blocker:** Cannot fetch journal entry line items via NetSuite REST API

### Root Cause Analysis

**NetSuite has TWO APIs that could work:**

#### 1. REST Record API (Tried, Blocked)

**Endpoint:** `GET /services/rest/record/v1/journalEntry/{id}?expandSubResources=true`

**Problem:** User Event Script executes on EVERY call and crashes:
```json
{
  "error": "SSS_SEARCH_ERROR_OCCURRED",
  "message": "Record 'CUSTOMRECORD_VID_TEMPLATE' was not found",
  "stack": "VID_DistributionTemplateGateway.runQuery(BaseGateway.js:305)"
}
```

**Script Details:**
- Name: TD UE VendorBillForm
- Deployment: CUSTOMDEPLOY_VID_UE_JOURNAL_VBF (you said this doesn't exist in deployments list)
- Type: User Event Script
- Bundle: com.netsuite.vendorinvoicedistribution

**Impact:** Blocks 100% of journal entry detail fetches

**Evidence:**
- Tested with 395 different journal entry IDs
- ALL returned the same script error
- The custom record DOES exist in your NetSuite
- But the script's query syntax or permissions are wrong

#### 2. SuiteQL API (Tried, SQL Syntax Error)

**Endpoint:** `POST /services/rest/query/v1/suiteql`

**Attempted Query:**
```sql
SELECT
    Transaction.id,
    Transaction.tranid,
    Transaction.trandate,
    Transaction.subsidiary,
    Transaction.status
FROM Transaction
WHERE Transaction.type = 'Journal'
```

**Error:** "Failed to parse SQL" - 400 Bad Request

**Reason:** SuiteQL has specific syntax requirements:
- Different from standard SQL
- Needs `BUILTIN_RESULT.TYPE_*` functions for type casting
- Column names may be different
- JOIN syntax is specific

**Status:** Partially implemented, needs NetSuite SuiteQL documentation to fix syntax

## Key Technical Details for Debugging

### Why 2,975 Bronze Records Are Useless

**Old API sync stored:**
```json
{
  "id": "7605",
  "links": [{"rel": "self", "href": "..."}]
}
```

**No financial data** - just metadata. Silver layer needs:
```json
{
  "id": "7605",
  "line": [
    {"account": {...}, "debit": 5000, "credit": 0},
    {"account": {...}, "debit": 0, "credit": 5000}
  ]
}
```

**Silver SQL:** `LATERAL FLATTEN(input => raw_data:line)`
- If `line` field doesn't exist → 0 rows produced
- **This is correct** - can't calculate financials without line items

### NetSuite REST API Discoveries

1. **List endpoints only return IDs:**
   ```
   GET /journalEntry?limit=100
   Returns: [{id, links}, {id, links}, ...]
   ```

2. **fields parameter rejected:**
   ```
   GET /journalEntry?fields=line,tranDate
   Error 400: "Use only q, limit, offset"
   ```

3. **expandSubResources only works on detail endpoint:**
   ```
   GET /journalEntry/{id}?expandSubResources=true
   Should return: {id, tranDate, line: [...]}
   But crashes with script error
   ```

### Two-Phase Sync (Implemented, Ready When API Fixed)

**Code location:** `mcp-server/src/services/snowflake_netsuite_loader.py` lines 241-296

**Logic:**
1. Phase 1: Fetch IDs from list endpoint (fast)
2. Phase 2: For each ID, fetch details with expandSubResources=true (slow but complete)
3. Store records with line items in Bronze

## Key Learnings & Gotchas

### NetSuite REST API Limitations

1. **List endpoints are metadata-only** - Only return IDs, must fetch details individually
2. **expandSubResources required** - Only way to get sublists via REST
3. **fields parameter not supported** - NetSuite rejects it with 400
4. **User Event Scripts execute on API calls** - Can block access if buggy
5. **SuiteQL is the workaround** - Bypasses User Event Scripts

### Snowflake Dynamic Tables

1. **Auto-refresh lag:** TARGET_LAG = '1 hour'
2. **Manual refresh:** `ALTER DYNAMIC TABLE [name] REFRESH`
3. **Dependency order:** Must refresh Silver before Gold
4. **LATERAL FLATTEN:** Requires source field to exist, otherwise produces 0 rows

### Circuit Breaker

**Current settings:** 20 failures, 300s (5min) recovery
**Reset:** Restart MCP container
**Check state:** Look for "Circuit breaker is OPEN" in logs
